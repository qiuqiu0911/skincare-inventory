# Cosmetics Tracker 微信小程序版

这是参照 `skincare-inventory-&-tracker` 前端原型实现的微信小程序版，采用原生小程序 WXML/WXSS/JS，不依赖云开发或第三方包。

## 功能范围

- 今日记录：早/晚分段查看，底部弹层新增或编辑记录，支持手写产品名和历史产品候选。
- 库存管理：使用中、囤货、已用完三段状态，支持新增、编辑、开瓶、标记用完和删除未开瓶库存。
- 回看统计：近 3 天早晚记录回顾，本周产品使用点阵概览。
- 分类管理：本地维护分类，下次记录和库存新增时可直接选择。
- 本地持久化：数据通过 `wx.setStorageSync` 保存在微信小程序本地存储中。
- 导入/导出：在“我的”页导出 JSON 备份文件，也可以从 JSON 文件导入恢复。

## 打开方式

## 数据持久化

当前默认不依赖微信云服务。记录、库存和分类会写入小程序本地 storage：

- 正常关闭小程序、切到后台、重新打开后，数据仍会保留。
- 不同设备之间不会自动同步。
- 卸载小程序、清理微信缓存、换设备，可能导致本地数据不可用。
- 建议定期在“我的”页导出 JSON 备份；导入会覆盖当前本机数据。

## 可选云开发同步

当前未开通微信云服务，云同步默认关闭。后续如需启用：

1. 在微信开发者工具中开通云开发。
2. 创建云数据库集合：`cosmetics_tracker_user_stores`。
3. 将集合权限设置为“仅创建者可读写”。
4. 在 `utils/cloudConfig.js` 中设置 `enabled: true` 并填入云环境 ID。

## 技术边界

- 当前版本没有自建账号、云同步、分享和订阅消息。
- 已预留微信云开发同步代码，但默认关闭。
- 数据层位于 `utils/store.js`，记录、库存、分类和统计均从同一份本地 store 派生。
- 免费个人小程序可以本地预览和真机调试；正式发布需在微信公众平台注册小程序。
